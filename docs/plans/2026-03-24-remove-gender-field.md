# Remove Gender Field from AIcall

## Problem

The `gender` field on AIcall is redundant. The TTS voice ID already determines voice characteristics (gender, accent, tone). Gender is threaded through the AIcall creation chain as a parameter but provides no value beyond what the voice ID already encodes.

## Scope

AIcall chain only. The TTS streaming `gender` field (bin-tts-manager) is out of scope — it is actively used by all three TTS providers (ElevenLabs, AWS, GCP) for voice ID fallback selection and will be addressed separately.

## Approach

Remove `gender` from the AIcall creation/start chain — model structs, function signatures, request models, RPC interfaces, OpenAPI schema, database column, and documentation. No backward compatibility — clean removal. Migration runs first, then deploy.
