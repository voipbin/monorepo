# CLAUDE.md and Docs Refresh — Design Document

**Date:** 2026-05-10
**Status:** Approved (4 critic review rounds)
**Scope:** All 37 services in the monorepo

---

## Goals

1. **Content accuracy** — service docs reflect actual code (real endpoints, correct patterns, current config)
2. **Structural consistency** — every service has the same predictable doc layout
3. **Depth** — four layers added: domain logic, inter-service dependencies, operational context, architecture detail
4. **Template-driven** — docs generated by reading actual code, not generic placeholders

---

## Design Decisions Summary

| Decision | Choice | Rationale |
|---|---|---|
| Per-service structure | Hybrid: slim CLAUDE.md + docs/*.md | CLAUDE.md stays as AI entry point; deep content in linked files |
| Doc file count | 4 standard files (varies by class) | Fixed layout → predictable navigation across 37 services |
| Generation method | Deterministic extractor → LLM prose | Extractor for tables/routes/config; LLM only for prose narratives |
| Root CLAUDE.md trim | Self-containment rule (not line count) | Any rule with a feedback_*.md entry stays inline; rationale not separated from rule |
| Service taxonomy | 7 classes | Services differ enough that a single template produces wrong output for ~7 of 37 |

## Service Taxonomy

Seven classes, assigned deterministically by filesystem inspection (`docs/reference/service-taxonomy-gen.sh`).

| Class | Count | Services | Doc shape |
|---|---|---|---|
| A | ~30 | bin-*-manager with `pkg/listenhandler/` | CLAUDE.md + docs/{architecture,domain,dependencies,operations}.md + optional subsystems.md |
| A2 | 2 | bin-sentinel-manager, bin-hook-manager | Same as A but no routing table (no listenhandler) |
| A+sub | 3 | voip-asterisk-proxy, voip-kamailio-proxy, voip-rtpengine-proxy | Class A + required subsystems.md (native daemon config) |
| B | 1 | bin-api-manager | CLAUDE.md + docs/{architecture,routing,auth,operations}.md |
| C | 1 | bin-common-handler | CLAUDE.md only (shared library, no docs/ subdir) |
| D | 1 | bin-dbscheme-manager | CLAUDE.md + docs/{migrations,schema-ownership,operations}.md |
| E | 1 | bin-openapi-manager | CLAUDE.md + docs/{generation,consumers}.md |

The taxonomy script is committed and asserted in CI. When a new service is added, CI fails until the script output is regenerated.

## Per-Service Doc Structure (Class A — the common case)

```
bin-<service>-manager/
├── CLAUDE.md                  ← slim index (~80-250 lines; CRITICAL block has no length cap)
└── docs/
    ├── architecture.md        ← Mermaid diagram, layer table, routing table (extractor-sourced)
    ├── domain.md              ← domain entities, state machines, business rules (LLM prose)
    ├── dependencies.md        ← machine-rendered from extractor JSON (no LLM, no hand-editing)
    ├── operations.md          ← failure modes, debugging, config table, metrics (mixed)
    ├── subsystems.md          ← (optional: non-Go embedded subsystems; required for A+sub)
    └── plans/                 ← (preserved; README.md added pointing to architecture.md)
```

### CLAUDE.md (index role)

- 2-3 sentence overview
- Key concepts list (3-5 bullets)
- Common commands: build, test, lint, generate (copy-paste ready)
- **Service-specific CRITICAL rules inline** (no length cap — safety > brevity)
- Links: `→ docs/architecture.md`, `→ docs/domain.md`, `→ docs/dependencies.md`, `→ docs/operations.md`
- One-liner: "Cross-cutting rules in root CLAUDE.md"

### docs/architecture.md

- Mermaid component diagram
- Layer responsibilities table
- Request routing table (sourced from extractor — deterministic)
- Core data models

### docs/domain.md

- Domain entities and relationships
- State machines (e.g., call: ring → up → hangup)
- Key business rules and invariants
- Important algorithms

### docs/dependencies.md (deterministic template render)

Rendered from `docs/reference/dependencies.md.tmpl` using extractor JSON. Never edited by hand. Diffs are always clean. Sections:
- Inbound callers (extracted)
- Outbound RPC targets (extracted from go.mod replace directives)
- Events subscribed (extracted from `cmd/*/main.go` subscribeTargets)
- Events published (extracted from pkg/**/go sources)
- go.mod replace directives (verbatim)
- WebhookMessage contracts → link to `bin-api-manager/docsdev/source/` only, no field list restated here

### docs/operations.md

- Common failure modes: symptom → probable cause → fix
- Debugging runbook: kubectl, RabbitMQ inspection, log grep patterns
- Known gotchas (service-specific)
- Config table (extractor-sourced — deterministic)
- Prometheus metrics table (extractor-sourced — deterministic)

## Root CLAUDE.md Restructure

### Trim Criterion

**Any CRITICAL section with a corresponding `feedback_*.md` memory entry stays inline and self-contained.** Rule + rationale together, never split across files. This is principled: inline text reflects documented prior incidents.

### Sections Confirmed to Stay Inline

- Worktrees rule + rationale
- Verification workflow table + rationale (`feedback_verification_bulk_changes.md`)
- Never push to main
- Squash merge + rationale (`feedback_squash_merge_required.md`)
- Alembic rule + rationale (`feedback_alembic_revision_ids.md`, `feedback_db_tag_requires_migration.md`)
- RST docs sync + rationale (`feedback_rst_docs_always.md`)
- Conflict check before PR/merge
- "Where to Document New Information" decision tree (routing rule, must be inline)
- "Where to find things" index

### What Moves Out

- Commit/PR message format examples → `docs/workflows/git-workflow-guide.md`
- API testing guidelines → `monitoring/CLAUDE.md`
- Grafana dashboard convention → `monitoring/CLAUDE.md`
- Quick links reference block → README.md pointer

**Expected result:** ~300-350 lines (no hard lower bound; self-containment rule takes precedence over brevity).

## Extractor Contract

### Tool

`grep`/`find` pipelines on raw Go source text. No ast-grep required (not installed). Fallback: `go/parser` for cases requiring symbol resolution.

### Output Location

`docs/.docs-gen/<service-name>.extracted.json` — centralized, single `.gitignore` entry, no per-service pollution. Committed during pilot for human review; gitignored in batch run.

### JSON Schema (schema_version: "1.0")

```json
{
  "schema_version": "1.0",
  "service_name": "bin-call-manager",
  "service_class": "A",
  "routing_table": [
    {"variable_name": "regV1Calls", "pattern": "/v1/calls$", "http_method": "POST"}
  ],
  "events_subscribed": [
    {"queue_symbol": "commonoutline.QueueNameAsteriskEventAll"}
  ],
  "events_published": [
    {"file": "pkg/callhandler/action.go", "event_type_symbol": "call.EventTypeCallCreated"}
  ],
  "dependencies": [
    {"module_path": "monorepo/bin-common-handler", "local_path": "../bin-common-handler"}
  ],
  "config_vars": [
    {"flag": "database_dsn", "env": "DATABASE_DSN", "type": "string", "required": true, "default": "", "description": "Database connection DSN"}
  ],
  "metrics": [
    {"name": "call_manager_receive_request_process_time", "type": "histogram", "labels": ["type", "method"]}
  ],
  "missing_fields": []
}
```

### Verified Extraction Algorithms

**1. Routing table** — `pkg/listenhandler/main.go`:
```bash
grep -n 'regexp\.MustCompile' pkg/listenhandler/main.go
grep -n '\.MatchString\|http\.Method' pkg/listenhandler/main.go
```
Output: `{variable_name, pattern_string, http_method}` per route.

**2. Events subscribed** — `cmd/*/main.go`:
```bash
grep -n -A 30 'subscribeTargets\s*:?=\s*\[\]string{' cmd/*/main.go
```
Capture string literal values (symbolic constants emitted as-is).

**3. Events published** — all `pkg/**/*.go` + `internal/**/*.go` (verified: calls live in callhandler, dbhandler, and other pkg subdirs):
```bash
find pkg internal -name "*.go" ! -name "*_test.go" ! -name "mock_*.go" 2>/dev/null \
  | xargs grep -n 'PublishWebhookEvent\|CallPublishEvent'
```
Output: `{file, line, event_type_symbol}`.

**4. Dependencies** — `go.mod` replace directives (per-service, never global):
```bash
grep -n '^replace ' go.mod
```

**5. Config vars** — `internal/config/*.go` preferred, fallback `cmd/*/init.go`:
```bash
CONFIG_SRC=$(ls internal/config/*.go 2>/dev/null | head -1 || ls cmd/*/init.go 2>/dev/null | head -1)
grep -n 'f\.String\|f\.Int\|f\.Bool\|f\.StringSlice' "$CONFIG_SRC"
# env var bindings live in a separate map:
grep -n '"[a-z_]*":\s*"[A-Z_]*"' "$CONFIG_SRC"
```

**6. Prometheus metrics** — all Go files:
```bash
grep -rn 'prometheus\.NewHistogramVec\|prometheus\.NewCounterVec\|prometheus\.NewGaugeVec\|prometheus\.NewSummaryVec' \
  --include="*.go"
```

**Missing field behavior:** Empty array → add field name to `missing_fields`. Never halt generation. Class A2 always has empty `routing_table`; not counted as missing.

## Generation Pipeline

### Phase 0 — Commit prerequisites (before any service work)

1. Rewrite `docs/reference/claude-md-template.md` (new 4-file structure + class variants)
2. Commit `docs/reference/service-taxonomy-gen.sh` (class-assignment script)
3. Commit `docs/reference/service-taxonomy.md` (script output, CI-asserted)
4. Commit `docs/reference/dependencies.md.tmpl` (deterministic template for dependencies.md)
5. Update root CLAUDE.md: "34 Go microservices" → "37 services" (cosmetic)
6. Add `docs/.docs-gen/` to root `.gitignore`

### Phase 0.5 — Class-assignment script

```bash
#!/usr/bin/env bash
# Order matters — each test eliminates by filesystem ground truth, in priority order

for svc_dir in bin-*/  voip-*/; do
  name=$(basename "$svc_dir")

  # Class D: Python/Alembic (alembic.ini present)
  if [ -f "$svc_dir/alembic.ini" ]; then echo "$name: D"; continue; fi

  # Class E: OpenAPI codegen (no cmd/, has openapi/)
  if [ ! -d "$svc_dir/cmd" ] && [ -d "$svc_dir/openapi" ]; then echo "$name: E"; continue; fi

  # Class C: shared library (no cmd/, has pkg/)
  if [ ! -d "$svc_dir/cmd" ] && [ -d "$svc_dir/pkg" ]; then echo "$name: C"; continue; fi

  # Class B: API gateway (has cmd/ but no listenhandler; uses oapi-codegen)
  if [ -d "$svc_dir/cmd" ] && [ ! -d "$svc_dir/pkg/listenhandler" ] && \
     grep -q "oapi-codegen" "$svc_dir/go.mod" 2>/dev/null; then
    echo "$name: B"; continue
  fi

  # Class A2: event-driven worker (has cmd/, no listenhandler, not B)
  if [ -d "$svc_dir/cmd" ] && [ ! -d "$svc_dir/pkg/listenhandler" ]; then
    echo "$name: A2"; continue
  fi

  # Class A+sub: voip-* proxies with listenhandler (Go + embedded native daemon)
  if [ -d "$svc_dir/pkg/listenhandler" ] && [[ "$name" == voip-* ]]; then
    echo "$name: A+sub"; continue
  fi

  # Class A: standard Go RPC manager
  if [ -d "$svc_dir/pkg/listenhandler" ]; then
    echo "$name: A"; continue
  fi

  echo "$name: UNKNOWN"
done
```

Self-test assertion after generating taxonomy: `grep "bin-api-manager: B"` must succeed.

### Phase 1 — Inventory

Walk all service dirs. Enumerate all existing `doc/`, `docs/`, `docsdev/` dirs. Classify:
- `bin-call-manager/doc/` → RST files, do not touch
- `bin-api-manager/docsdev/source/` → RST source, do not touch
- `*/docs/plans/` → preserved, add one-line README

Commit output to `docs/reference/service-inventory.md`.

### Phase 2 — Extractor run

Run extractor on all 37 services. Output to `docs/.docs-gen/<service>.extracted.json`. Log all `missing_fields`.

### Phase 2.5 — Coverage audit

Compute field population rate per class. **Threshold: ≥85% of applicable fields populated.** (Class A2 `routing_table` is N/A; Class D/E have no Go extraction.) Commit `docs/reference/extractor-coverage-report.md`. Block Phase 3 if threshold not met.

### Phase 3 — Pilot (5 services, human review before scale-out)

1. bin-call-manager (Class A — complex)
2. bin-api-manager (Class B — gateway)
3. bin-dbscheme-manager (Class D — Python)
4. bin-sentinel-manager (Class A2 — event-driven)
5. voip-asterisk-proxy (Class A+sub — Go + native daemon)

During pilot: commit extractor JSON for all 5 to `docs/.docs-gen/`. Run full generation. Human reviews structure, accuracy, and prose quality. Derive required-section-heading checklists. Update template if needed before batch run.

**Collision rule:** Generate new files. Preserve `docs/plans/` and contents. Overwrite any pre-existing `docs/architecture.md` etc. (none currently exist).

### Phase 4 — Batch generation (post-pilot approval)

Batches of 5 services. Per-batch commit. **6-field spot-check on 2 randomly selected services per batch:**
1. Routing table: compare 3 random routes to `pkg/listenhandler/main.go`
2. Events subscribed: compare to `cmd/*/main.go` subscribeTargets
3. Events published: grep `pkg/**/go` sources and compare
4. Dependencies: compare to `go.mod` replace directives verbatim
5. Config vars: compare 3 random vars to `internal/config/*.go`
6. Metrics: compare to `prometheus.New*` calls

Log spot-check in PR description. Block merge if any mismatch.

### Phase 5 — Root CLAUDE.md update

Apply the trim described above (~300-350 lines expected).

### Phase 6 — Verification

1. `markdown-link-check` on all new `docs/*.md` files
2. `git diff --name-only | grep -E 'docsdev/source|docsdev/build'` must be empty
3. Spot-check config tables (3 random services) against `internal/config/*.go`

## Constraints

- `doc/` (singular) and `docsdev/source/`: never modified
- `docs/plans/` in any service: preserved; `plans/README.md` added
- Root `docs/` tree (architecture/, conventions/, etc.): unchanged by this pass
- No new files at monorepo root level
- `bin-common-handler`: CLAUDE.md only, no `docs/` subdir
- Extractor JSON: committed during pilot, gitignored in batch run

---

## Rollback

Each batch is a separate commit (squash-merged per project convention). Rollback: `git revert <batch-sha>` on main, then re-run that batch after fixing the issue.
